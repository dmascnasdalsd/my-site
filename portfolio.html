恋⼼智能陪伴助⼿

基于 Spring Boot 3 + Spring AI 的企业级 AI 陪伴助手

技术架构

Spring Boot 3 Spring AI PGvector Redis Ollama Docker 阿里云百炼

难点1：RAG知识库检索精准度低

问题描述

文档切片不合理、向量存储匹配度不足，导致AI检索时无法精准匹配用户查询意图，出现回复偏离、无关信息过多的情况，影响用户体验。

解决方法

1

优化文档切片策略

基于TokenTextSplitter自定义切片规则，配置合理的分块大小和重叠比例，确保文本语义完整性，避免切片过细或过粗导致的语义断裂。

2

增强向量检索能力

结合KeywordMetadataEnricher实现AI智能关键词提取，为文档添加元信息标注（如场景标签、核心关键词），通过FilterExpressionBuilder构造过滤规则，实现精准筛选。

3

优化查询策略

集成MultiQueryExpander多查询扩展、RewriteQueryTransformer查询重写机制，将用户原始查询扩展为多个语义相关变体，提升检索召回率和精准度。

4

调试优化参数

通过反复测试，调整PGvector向量数据库的相似度阈值、返回文档数量，平衡检索速度和精准度。

难点2：AI智能体易陷入Agent Loop死循环

问题描述

基于ReAct模式构建的自主规划智能体，在处理复杂任务（如多步骤工具调用、任务分解）时，容易出现重复执行同一操作、无法判断任务是否完成的情况，导致系统资源浪费、响应超时。

解决方法

1

增加死循环检测机制

设置任务执行最大步骤限制，记录智能体每一步的执行状态，若出现重复操作或步骤超时，自动触发中断逻辑。

2

开发终止工具

自定义终止工具，通过Prompt工程优化智能体判断逻辑，让AI能够自主识别任务完成条件（如目标达成、无可用工具），主动调用终止工具结束流程。

3

优化任务规划逻辑

参考OpenManus源码，实现分层智能体架构（BaseAgent、ReActAgent），明确任务分解、工具选择、结果校验的流程，减少无效循环。

4

增加状态管理

维护智能体执行状态（待执行、执行中、已完成、异常），每一步执行后校验结果，确保任务按预期推进。

项目亮点

技术前沿

聚焦AI超级智能体核心技术，整合Spring AI、RAG、Tool Calling、MCP等前沿技术，实现自主规划、多工具调用、多模型适配，贴合当前AI应用开发的主流方向。

核心突破

针对RAG检索精准度、Agent Loop死循环等核心难点，设计可落地的解决方案，将RAG检索精准度提升30%以上，解决智能体死循环问题，确保系统稳定性和用户体验。

商惠通优惠券运营平台

基于 SpringBoot3 + 微服务架构的高并发优惠券营销系统

技术架构

SpringBoot3 SpringCloudAlibaba Nacos Redis RocketMQ 5.x MySQL ShardingSphere Sentinel XXL Job Redisson ElasticSearch

难点1：高并发场景下优惠券分发漏发、多发，库存一致性难以保障

问题描述

项目需支持百万级别用户优惠券分发，高峰期（如秒杀、大额券推送）并发请求量激增，易出现库存扣减异常、优惠券漏发、多发，以及分布式场景下库存数据不一致的问题，严重影响业务准确性和用户体验。

解决方法

1

采用MySQL悲观行记录锁

在优惠券分发过程中，通过悲观锁锁定库存记录，确保库存不会被多扣，同时搭配自旋重试机制，直至库存扣减至零，避免并发扣减异常。

2

实现幂等性控制

在用户优惠券表创建唯一索引，限制用户单张优惠券只能领取一次，发放时捕获数据库重复记录异常，采用乐观自旋机制反复尝试，确保不重复发放。

3

优化分发流程

通过执行点位和批量处理优化推送业务，保障项目断电后可从上次执行位置继续分发，避免漏发，同时提升分发效率，适配高并发场景。

4

消息队列兜底

利用RocketMQ 5.x解耦优惠券推送与通知逻辑，通过消息重试机制，确保消息不丢失，进一步保障分发的准确性。

难点2：大数据量处理导致内存溢出、响应缓慢

问题描述

商家推送优惠券时，需解析百万量级用户Excel文件，易出现内存溢出；用户查询可用/不可用优惠券列表时，涉及多表关联、大量数据查询，导致接口响应缓慢，影响用户体验。

解决方法

1

优化Excel解析

使用EasyExcel解析百万量级Excel文件，采用分片读取、异步处理方式，避免因文件过大导致的内存溢出问题。

2

异步提升响应速度

通过ThreadPoolExecutor线程池和Redisson延时队列，异步执行Excel文件行数解析，降低优惠券推送接口的响应时间。

3

缓存优化查询性能

使用Redis Pipeline管道命令获取优惠券模板详情，将查询性能提升一倍以上，适配大量用户查询场景。

4

分库分表扩容

采用ShardingSphere分库分表方案，存储大量商家创建的优惠券模板数据，提升存储和查询效率，应对大数据量存储需求。

难点3：缓存异常及缓存与数据库数据一致性问题

问题描述

优惠券查询、用户预约提醒等场景频繁依赖缓存，易出现缓存击穿、穿透问题，导致大量请求直接打向数据库，造成数据库压力激增；同时存在数据库数据更新后，缓存未及时同步，出现数据不一致的情况。

解决方法

1

防御缓存异常

采用布隆过滤器、缓存空值、分布式锁三重方案，防止恶意请求导致的缓存击穿和穿透，减少数据库压力。

2

保障数据一致性

基于监听Binlog机制，异步同步数据库更新至Redis缓存，同时采用“写后查询”策略，应对Redis持久化或主从复制极端情况下的数据丢失问题，确保缓存与数据库数据一致。

3

缓存结构优化

基于Redis ZSet数据结构存储用户已领取优惠券列表，以领取时间为Score进行倒序排序，既提升查询效率，又满足业务展示需求。

4

分布式锁控制

使用Redisson分布式锁，保障优惠券锁定、核销、退款等操作的同步执行，避免缓存与数据库操作冲突。

项目亮点

高并发高可用

针对优惠券分发、秒杀等核心业务，设计悲观锁、幂等性控制、消息队列重试等方案，解决高并发下库存不一致、漏发多发问题，可稳定支撑百万级用户优惠券分发，保障系统高可用。

细节打磨到位

针对缓存异常、重复请求、数据一致性等细节问题，设计多重防御方案；通过操作日志留痕、布隆过滤器、分布式锁等技术，确保业务操作可追溯、数据安全可靠，避免业务风险。
